## The Problem

Small service businesses lose hours every week to back-and-forth scheduling. Customers need a simple way to see availability and book without calling — providers need bookings stored reliably without double-booking.

## The Solution

A provider directory with a 3-step booking flow: pick a date, choose a time slot, enter contact details. The system generates 30-minute slots from each provider's availability windows, skips already-booked times, and sends a confirmation email with a Google Calendar link.

## Key Features

- **Provider directory** — browse available service providers with name and bio
- **3-step booking flow** — date picker → slot selection → contact details
- **Smart slot generation** — 30-minute slots derived from provider availability windows, with double-booking prevention
- **Booking confirmation** — summary page with a one-click Google Calendar link
- **Email notifications** — Resend-powered confirmation emails (skippable via env var)
- **Demo mode** — persistent banner and client-side fake booking flow for portfolio deployments
- **Static export** — builds to fully static HTML for GitHub Pages or any CDN

## Technical Decisions

Slot generation is pure logic in `lib/slots.ts` — given a provider's availability windows and existing bookings, it returns the open 30-minute intervals for a given day. This keeps the booking UI fast and testable without hitting the database on every render.

The demo deployment uses Next.js static export with `NEXT_PUBLIC_DEMO_BANNER=true`, so the booking flow runs entirely client-side without a real database. The same codebase runs against a live PostgreSQL database (via Prisma) in a production deployment.
