## The Problem

Small service businesses — salons, physiotherapists, repair shops — lose hours every week to scheduling phone calls, no-shows, and double-bookings. Most existing tools are overpriced or too complex for a 5-person team.

## The Solution

A focused, multi-tenant booking platform where each business gets their own workspace. Clients book through a public-facing page; staff manage availability through a simple dashboard.

## Key Features

- **Multi-tenant architecture** — one codebase, isolated data per business via tenant ID
- **Public booking page** — clients pick a service, staff member, and time slot; receive automated email reminders
- **Staff schedule management** — set working hours, block off days, handle walk-ins
- **Stripe integration** — optional deposit collection at booking time
- **Automated reminders** — Resend-powered email confirmation and 24h reminders

## Technical Decisions

Tenant isolation is handled at the database level with a `tenant_id` foreign key on every resource table, enforced by row-level security in PostgreSQL. Each request passes through middleware that resolves the tenant from the subdomain and attaches it to the request context.

Stripe Checkout is used for deposits — no card data touches the server.
